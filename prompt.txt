1. Presentación y Contexto General
1.1 Introducción del Proyecto
Sistema Web sobre una Bibliotecaria en Digital
Descripción:
El Sistema Web de Biblioteca Digital es una plataforma en línea que permite gestionar libros en formato digital, autores, categorías y editoriales, así como el registro y control de usuarios.
El sistema permite a los usuarios registrarse, iniciar sesión, buscar libros, descargarlos, calificarlos y dejar reseñas.
Además, cuenta con un módulo administrativo para gestionar el catálogo digital, controlar la disponibilidad de libros, administrar usuarios y registrar acciones mediante un sistema de logs.
En términos generales, el sistema centraliza la gestión bibliográfica y facilita el acceso a contenido digital de manera organizada y segura.
Problema que resuelve:
Actualmente, muchas bibliotecas o instituciones educativas presentan los siguientes problemas:
Gestión manual o desorganizada de libros digitales.


Dificultad para controlar descargas y disponibilidad.


Falta de seguimiento sobre qué usuarios descargan qué libros.


Ausencia de un sistema estructurado para reseñas y calificaciones.


Poco control administrativo sobre el catálogo.
Esto genera:
Desorden en la información.


Pérdida de tiempo en búsquedas.


Falta de trazabilidad (no saber quién hizo qué).


Mala experiencia para los usuarios.
El sistema resuelve estos problemas mediante una plataforma centralizada que organiza la información, automatiza procesos y mejora la experiencia de acceso a libros digitales.
Usuarios objetivo: 
Usuarios Registrados
Buscar libros por título, categoría o autor.


Descargar libros digitales.

4.2 Aplicación de Conceptos Avanzados
La implementación del sistema se desarrolló siguiendo el paradigma de Programación Orientada a Objetos (POO), que es un modelo de programación basado en la creación de objetos que contienen datos y comportamientos.
Se aplicaron los siguientes principios:
Separación de responsabilidades


Encapsulamiento


Bajo acoplamiento


Alta cohesión
Herencia
La herencia permite que una clase hija reutilice atributos y métodos de una clase padre.
En el sistema:
Los repositorios específicos (UserRepository, BookRepository) heredan de BaseRepository.


Esto evita duplicación de lógica de conexión y operaciones comunes.


Beneficio:
Reutilización de código


Organización jerárquica


Reducción de redundancia
Interfaces
Una interfaz define un conjunto de métodos que una clase está obligada a implementar.
En el sistema:
IRepository define las operaciones básicas como buscar o eliminar.
Todos los repositorios deben cumplir ese contrato.


Beneficio:
Uniformidad en la arquitectura
Flexibilidad para cambiar implementaciones
Facilita pruebas unitarias
Clases Abstractas
Una clase abstracta es una clase base que no puede instanciarse directamente y sirve como plantilla para otras clases.
En el sistema:
BaseRepository centraliza lógica común.
BaseException sirve como base para excepciones específicas.
Beneficio:
Define estructura común
Permite extender comportamiento sin repetir código
Polimorfismo
El polimorfismo permite que diferentes clases respondan al mismo método de distintas maneras.
En el sistema:
UserService trabaja con la interfaz IRepository.
Puede recibir cualquier clase que implemente ese contrato.
No depende directamente de una implementación concreta.
Beneficio:
Bajo acoplamiento
Mayor escalabilidad
Código más flexible
4.3 Manejo de Excepciones y Errores
Uso de bloques try-catch
El bloque try-catch permite capturar errores y manejarlos de forma controlada sin detener completamente la ejecución del sistema.
Aplicación en el sistema:
Captura errores cuando no se encuentra un registro.


Controla fallos de base de datos.


Evita que el sistema muestre errores técnicos al usuario final.


Beneficio:
Mayor estabilidad


Mejor experiencia de usuario


Control centralizado de errores
Excepciones Personalizadas
Una excepción personalizada es una clase que representa un error específico del dominio del sistema.
Ejemplo conceptual:
UserNotFoundException representa el caso en que un usuario no existe.
Ventajas:
Permite diferenciar tipos de error.
Mejora la claridad del código.
Facilita depuración y mantenimiento.
Sistema de Logging
El logging es el registro sistemático de eventos importantes dentro de una aplicación.
En el sistema se registran:
Errores críticos


Intentos fallidos de acceso


Operaciones importantes


Beneficios:
Auditoría del sistema


Detección de fallos


Monitoreo y mantenimiento


Soporte técnico más eficiente
